A struct is identical to a [[Classes|class]], except it is a [[Value Types vs Reference Types#Value types|value type]]. 95% of the time you will use a [[Classes|class]], so here's just how to define one and a table with the differences between them.
# Defining a struct
```C#
struct ThisIsMyStructNameForRealz
{

}
```
# Difference between class and struct

| Characteristic           | Structs                                                                                                                                                                                                   | Classes                                                                                                                                                                |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Type**                 | Value types, stored on the stack                                                                                                                                                                          | Reference types, stored on the heap with a reference on the stack                                                                                                      |
| **Memory Allocation**    | Faster allocation/deallocation for small data structures                                                                                                                                                  | Have overhead but offer more flexibility for complex objects                                                                                                           |
| **Inheritance**          | Cannot inherit from other structs or classes (can only implement interfaces)                                                                                                                              | Support inheritance, allowing for class hierarchies                                                                                                                    |
| **Assignment Behavior**  | Create a copy of all data when assigned to a new variable                                                                                                                                                 | Copy only the reference; both variables point to the same object                                                                                                       |
| **Default Constructors** | Have implicit parameterless constructors that initialize fields to default values                                                                                                                         | Require explicit parameterless constructors if needed                                                                                                                  |
| **Best Used When**       | Creating small, simple data structures<br>Object's lifetime is short<br>Object will not be modified after creation<br>Value semantics (copying on assignment) is desired<br>Memory efficiency is critical | Creating complex objects with behaviors<br>Inheritance is needed<br>Object needs reference semantics<br>Object will be modified after creation<br>Object might be null |
| **Examples**             | Point coordinates, dates, colors, small mathematical objects                                                                                                                                              | Business entities, services, controllers, UI components                                                                                                                |
